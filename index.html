How it works

Move between a template and the code it creates.

01

Understand template parts

JetForge recognizes directives, Java blocks, expressions, includes, and plain output text.

02

Map source to output

Preview ranges connect template lines to generated code, so reveal commands land in the right place.

03

Keep Java help available

Completions, hover, Go to Definition, and local fallbacks keep working inside template Java.

04

Generate and compare

Run local generation, then compare the latest output with the previous preview.

Features

Editor help for the JET details that are easy to lose.

Catch template mistakes

Find unclosed blocks, unexpected delimiters, malformed directives, invalid attributes, and missing includes.

Jump to related files

Go to includes, skeletons, helper declarations, output previews, and generated Java regions.

Format safely

Clean up directive attributes, expression spacing, and Java blocks without rewriting output text.

Stay local

No runtime network access, no telemetry, no template uploads, and no proprietary code leaving VS Code.

Target modes

Choose the language your template outputs.

Use Java, HTML, XML, C, or Python output previews while embedded Java remains readable and navigable.

Local install

Build and install the extension locally.

$ npm run verify
